#### 关于指标到获取数据的映射方式

在ABILITY_INDICATOR_MAPPING表中需要设置一个extra_param字段用于存储一个JSON类型的信息，这个信息用于获取数据时所需要的额外约束字段`{param1: value1, param2: value2}`，即访问数据库时的 `where param1 = value1 and param2 = value2`

![[Pasted image 20260709152011.png | 1000]]

1. 主跑道数量：
   跑道表中存在标记该跑道类型的字段（0-主跑道、1-备用跑道）
	1. table: 跑道表
	2. oper: count * from 跑道表 where type = 0
	3. extra_param: {type: 0}
3. 跑道长度：
   一般衡量机场能力时，构建指标取最长的跑道长度作为指标输入（不考虑主还是备用跑道）
	1. table: 跑道表
	2. oper: max 长度 from 跑道表
4. 主滑行道数量：
   滑行道表中存在标记该滑行道类型的字段（0-主滑行道、1-备用滑行道）
	1. table: 滑行道表
	2. oper: count * from 滑行道表 where type = 0
	3. extra_param: {type: 0}
5. 防空最大射程：
   以机场内部署的射程最大的防空设施的射程作为该机场的防空最大射程
	1. 表：防空设施表
	2. oper: select max(射程) from 防空设施表
6. 防空最大射高：
   以机场内部署的射高最大的防空设施的射程作为该机场的防空最大射高
	1. table: 防空设施表
	2. oper: select max(射高) from 防空设施表
