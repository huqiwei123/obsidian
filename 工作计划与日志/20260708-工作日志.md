### 情报结果页的核心用户需求

1. **一眼看总体态势**：八个机场整体能力水平如何，是否存在明显强弱分化。
2. **快速找重点**：哪些机场综合能力最强/最弱，哪些能力维度普遍偏弱。
3. **横向可对比**：同一能力下八个机场如何排序；同一机场的八项能力是否均衡。
4. **结果可解释**：用户点击某个分值后，可以看到影响该能力的关键维度、指标和原始目标数据来源。
5. **支持汇报输出**：页面不仅展示数据，还要能形成“情报结论摘要”。

## 方案一：总览驾驶舱 + 八能力热力矩阵（推荐）

### 页面结构

1. **顶部结论区**
   - 综合能力最强机场
   - 综合能力最弱机场
   - 整体最强能力维度
   - 整体最薄弱能力维度
   - 异常/差异最大能力维度

2. **核心区域：8 × 8 能力热力矩阵**
   - 行：八个机场
   - 列：八项能力：作战范围、备用接替、出动、打击、防空、伪装防护、通信保障、抢修恢复
   - 单元格：显示能力分值 + 等级颜色
     - 高：深色/绿色
     - 中：黄色/蓝色
     - 低：红色/浅色警示
   - 支持按机场总分排序、按某项能力排序、只看低能力项。

3. **右侧情报摘要栏**
   - 自动生成若干条结论，例如：
     - “A 机场综合能力最高，但通信保障能力相对偏弱。”
     - “B 机场防空能力排名第一，主要受防空最大射程和防空设施数量影响。”
     - “八个机场中，抢修恢复能力整体均值最低，应作为后续重点关注方向。”

4. **底部辅助图表**
   - 八项能力平均分柱状图
   - 八个机场综合得分排名条形图

### 优点

- 最符合“汇总结果页”的定位。
- 用户可以快速发现强弱、排名和异常。
- 热力矩阵非常适合展示“八个对象 × 八个指标”的结果。
- 后续可以自然扩展点击下钻能力详情。

### 适用场景

- 情报汇报首页
- 领导快速查看
- 分析人员快速定位重点机场和重点能力

### 建议交互

- 点击某个单元格：打开“机场 + 能力”详情抽屉，展示该能力的二级维度分值、关键指标、影响因素。
- 点击某个能力列头：进入该能力下八个机场的排名视图。
- 点击某个机场名称：进入该机场八项能力画像。

## 方案二：综合排名表 + 能力雷达概览

### 页面结构

1. **左侧/上方综合排名表**
   - 机场名称
   - 综合得分
   - 综合等级
   - 八项能力小型进度条
   - 优势能力
   - 短板能力

2. **右侧雷达图区域**
   - 默认展示排名第一机场的八能力雷达图。
   - 支持选择 2-3 个机场叠加对比。

3. **底部结论卡片**
   - 最强机场
   - 短板最多机场
   - 能力最均衡机场
   - 能力波动最大机场

### 优点

- 排名逻辑清晰，适合用户先看“谁强谁弱”。
- 雷达图便于观察单个机场的能力均衡性。
- 适合从“综合实力”角度做汇总。

### 不足

- 雷达图叠加过多时会混乱，最多建议同时对比 2-3 个机场。
- 不如热力矩阵直观展示全部 64 个分值。

### 适用场景

- 用户更关心综合排名。
- 页面希望偏表格化、数据化。

## 方案三：八个机场画像卡片墙

### 页面结构

1. **顶部整体摘要**

   - 高/中/低能力机场数量统计

2. **主体：八个机场卡片**
   - 每个机场一张卡片。
   - 卡片内容包括：
     - 综合得分和等级
     - 八项能力迷你条形图
     - 最高能力
     - 最低能力
     - 关键风险标签

3. **卡片排序方式**
   - 按综合得分排序
   - 按短板数量排序

### 优点

- 若用户要比较某一项能力下八个机场的差异，还需要额外图表辅助。

### 适用场景

## 方案四：能力维度分组汇总页

### 页面结构

1. **顶部能力 Tab / 分段按钮**

   - 备用接替能力
   - 出动能力
   - 打击能力
   - 防空能力
   - 伪装防护能力
   - 通信保障能力
   - 抢修恢复能力

2. **每个能力页展示**
   - 八个机场该能力得分排名
   - 该能力平均分、最高分、最低分、差值
   - 该能力下的二级维度对比
   - 影响该能力的关键指标 Top 3

   - 页面默认先展示“全部能力总览”。
   - 用户点击某项能力后进入能力专题分析。

### 优点

### 适用场景

- 后续要支持能力专题报告。

## 方案五：地图态势 + 汇总对比面板

### 页面结构

1. **左侧/主体地图**
   - 八个机场以点位展示。
   - 点位颜色表示综合等级。
   - 点位大小表示综合得分或目标规模。

2. **右侧汇总面板**
   - 当前选中能力的机场排名。
   - 八项能力切换按钮。
   - 综合态势结论。

3. **底部热力矩阵/排名条**
   - 补充展示 8 × 8 能力结果。

### 优点

- 空间感强，适合军事机场类场景。
- 能把“地理位置 + 能力强弱”结合起来。
- 更有态势感和汇报感。

### 不足

- 如果地图占比过大，能力对比信息会被弱化。
- 仍需要热力矩阵或排名表辅助表达具体分值。

### 适用场景

- 用户需要结合地理态势判断机场能力。
- 页面面向大屏展示、态势展示或指挥类场景。

## 方案六：情报结论流 + 可解释下钻

### 页面结构

1. **顶部一句话总评**
   - 例如：“八个机场整体呈现防空能力较强、抢修恢复能力偏弱的特征，A/B 机场综合能力领先。”

2. **主体：情报结论卡片流**
   - 综合能力结论
   - 优势机场结论
   - 短板机场结论
   - 能力异常结论
   - 能力均衡性结论
   - 后续关注建议

3. **每张结论卡片包含**
   - 结论标题
   - 支撑数据
   - 涉及机场
   - 涉及能力
   - 可点击查看证据链

4. **底部数据支撑区**
   - 热力矩阵
   - 排名表
   - 能力均值图

### 优点

- 最符合“情报结果”的语义，不只是展示数据，而是输出判断。
- 适合非专业用户、领导用户快速阅读。
- 可以把模型结果转化为可读结论。

### 不足

- 需要后端或前端增加结论生成规则。
- 如果只做结论、不展示数据支撑，可信度会不足。

### 适用场景

- 页面定位偏“最终研判结果”。
- 用户希望直接获得可汇报的情报摘要。

## 方案七：对比分析工作台

### 页面结构

1. **顶部多选机场 + 多选能力筛选器**
   - 默认选择全部八个机场、全部八项能力。
   - 用户可选择 2-4 个机场进行重点对比。

2. **中间对比图表**
   - 分组柱状图：对比多个机场在同一能力上的分值。
   - 雷达图：对比多个机场的八项能力形态。
   - 差值表：显示领先/落后幅度。

3. **右侧分析结论**
   - 选中机场之间的优势差异。
   - 能力短板差异。
   - 关键指标差异。

### 优点

- 适合分析人员做深入对比。
- 筛选自由度高。
- 能支持“选几个机场重点比较”的业务场景。

### 不足

- 页面复杂度较高。
- 对普通用户来说学习成本比总览页高。

### 适用场景

- 分析人员工作台。
- 需要频繁做机场之间的专项对比。

## 最终推荐

### 推荐采用：方案一为主，融合方案六和方案二的部分能力

建议情报结果页最终设计为：

1. **顶部：情报结论摘要**
   - 综合最强机场、最弱机场、最强能力、最弱能力、异常差异点。

2. **中部：8 × 8 能力热力矩阵**
   - 作为页面核心，用来直观展示八个机场八项能力的汇总结果。

3. **右侧/下方：综合排名与能力均值图**
   - 支撑用户快速看排名和整体能力结构。

4. **点击下钻：详情抽屉**
   - 点击某个机场/能力单元格，查看该能力的二级维度、关键指标和数据来源。

5. **结论输出：情报卡片**
   - 把数据结果转化成“可汇报”的情报判断。

### 推荐页面布局草图

```text
┌────────────────────────────────────────────────────────────┐
│ 情报结果总览                                               │
│ [综合最强] [综合最弱] [最强能力] [最弱能力] [异常差异]       │
├───────────────────────────────┬────────────────────────────┤
│ 8个机场 × 8项能力热力矩阵       │ 情报结论摘要                 │
│                               │ - 结论1                     │
│ 行：机场                       │ - 结论2                     │
│ 列：能力                       │ - 结论3                     │
│ 单元格：分值 + 等级颜色         │ - 后续关注建议               │
├───────────────────────────────┴────────────────────────────┤
│ 综合得分排名        八项能力平均分        短板能力统计        │
└────────────────────────────────────────────────────────────┘
```

### 页面设计关键点

- 不要继续以单个机场为主导航，应改为“总览 + 筛选 + 下钻”。
- 核心可视化优先选择热力矩阵，因为它最适合展示 64 个能力结果。
- 分值必须同时显示“数值 + 等级 + 颜色”，避免用户只看颜色无法判断差距。
- 汇总页一定要有结论区，否则只是数据看板，不像“情报结果”。
- 每个结论都要能回到支撑数据，增强可信度。

### 三个方案选择建议

| 方案 | 重点 | 优点 | 风险 |
|---|---|---|---|
| 方案一：热力矩阵驾驶舱 | 全局汇总与横向对比 | 最直观展示 64 个结果，最符合“情报结果汇总” | 需要做好颜色等级和排序交互 |
| 方案二：综合排名画像 | 机场排名与单机场画像 | 结构清晰，适合从机场维度分析 | 对“能力维度整体差异”的感知弱于热力矩阵 |
| 方案三：地图态势结论流 | 地理态势与情报研判 | 态势感强，适合汇报展示 | 地图可能抢占空间，仍需矩阵作为数据支撑 |

**最终建议：**
优先采用 **方案一：总览驾驶舱 + 八能力热力矩阵** 作为情报结果页主方案；如果后续需要增强汇报感，可以融合方案三的“情报结论流”；如果需要支持分析人员下钻，可以融合方案二的“选中机场能力画像”。
